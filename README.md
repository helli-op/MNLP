# Onboarding RAG-ассистент

 Каждая компания имеет уникальную корпоративную культуру и процессы. Новым сотрудникам сложно быстро вникнуть во все нюансы, а у руководителей и бадди нет ресурса времени на полноценный онбординг. Существующие wiki-страницы часто разрастаются, теряют структуру и становятся сложными для восприятия. Мы предлагаем  интеллектуального RAG-ассистента (Retrieval-Augmented Generation), который поможет новым и действующим сотрудникам быстро находить ответы на вопросы по корпоративным процессам, документации и регламентам. Ассистент самостоятельно ищет релевантную информацию в базе знаний и формирует понятные ответы.

## **Что было реализовано:**

1. Создание векторной БД с индексацией данных с помощью FAISS
2. Разбиение документов на чанки
3. Использование OpenRouter API для обращения к LLM-модели
4. Полный бэкенд (класс DocumentAssistant) с RAG-системой: поиск наиболее релевантных чанков
5. Бот в Telegram с пользовательской оценкой
6. Логирование запросов и оценок пользователей, а также ошибок

## Quick start

1. Создать env файл:

```python
cp .env.example .env
```

1. Установить зависимости:

```python
python3 -m pip install -r requirements.txt
```

1. Создание файла data.json с данными (названия файлов с документами):

```python
echo '["file1.docx", "file2.pdf", "file3.txt"]' > data.json
```

1. Запуск бота:

```python
python main.py
```

После запуска можно пользоваться ботом по ссылке, написав /start.
